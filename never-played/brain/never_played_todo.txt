## Long-term / Research

### 10. Steam Store API integration
**Benefits:**
- Game descriptions/synopsis
- Screenshots and media
- Detailed review breakdowns
- Price information
- More accurate metadata

**Trade-offs:**
- Slower load times (additional API calls)
- More complex caching logic
- Rate limiting concerns
- May need separate endpoint per game

**Implementation approach:**
- Start with on-demand fetching (only when user clicks game)
- Cache aggressively to avoid repeated calls
- Show loading state while fetching
- Graceful degradation if API fails

## Notes

**Priority Order:**
1. Fix bugs (#4, #6) - broken functionality hurts user experience
2. Add core features (#2 links, #3 played elsewhere) - high value, moderate effort
3. Polish UI (#7, #8) - improves usability
4. Research long-term (#10) - only if other features prove insufficient

**Development Philosophy:**
- Keep it simple - avoid feature bloat
- User value first - does this help people play games?
- Performance matters - fast is better than pretty
- Mobile-friendly - many users browse on phones

**Technical Debt to Watch:**
- SteamSpy rate limiting (currently 10 parallel requests)
- localStorage size limits (5-10MB)
